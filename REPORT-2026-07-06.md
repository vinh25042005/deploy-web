## 2. Lỗi gặp phải & cách sửa

| # | Lỗi | Nguyên nhân | Cách sửa |
|---|-----|------------|----------|
| 1 | Prometheus scrape `localhost:9100` fail | Prometheus container khác network với node-exporter | Cho Prometheus `network_mode: host` |
| 2 | Prometheus scrape `[::1]:9100` connection refused | `localhost` resolve ra IPv6 | Đổi target `localhost` → `127.0.0.1` |
| 3 | Grafana không resolve `prometheus:9090` | Prometheus host network → không còn Docker DNS | Đổi URL → `http://10.20.1.5:9090` |
| 4 | Loki `mkdir /var/loki` permission denied | Thiếu cấu hình path_prefix | Thêm `common.path_prefix: /loki` |
| 5 | Loki `mkdir wal` permission denied | Thiếu WAL dir config | Thêm `ingester.wal.dir: /loki/wal` |
| 6 | Grafana Loki datasource bị cache URL rỗng | Provisioning cũ không cập nhật | Reset toàn bộ monitoring, triển khai lại |
| 7 | Dashboard logs "no data" | JSON dashboard tham chiếu UID datasource sai | Bỏ UID cứng, chọn datasource khi import |
| 8 | Promtail label `hostname` không tự động | `relabel_configs` với `__host__` không hoạt động | Dùng static label `host: VM_PLACEHOLDER` + sed thay tên VM khi deploy |
| 9 | `docker compose` vs `docker-compose` | VM cài Docker Compose v1 | Dùng `docker-compose` (có dấu gạch ngang) |
| 10 | `host` network_mode xung đột `ports:` | Docker Compose v1 không cho phép ports mapping với host network | Bỏ `ports:` khỏi prometheus service |

## 4. Triển khai

Triển khai qua CI/CD (`deploy.yml`):

- Monitor: CI SCP `monitoring/` + `docker-compose up -d`
